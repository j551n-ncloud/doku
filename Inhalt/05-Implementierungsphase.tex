\section{Implementierungsphase}
\label{sec:Implementierungsphase}

\subsection{Beschaffung und Vorbereitung}
\label{sec:Beschaffung}
Da als Hardware ein vorhandener IBM System x3755 M3 nachgenutzt
wird, entfällt eine externe Beschaffung. Der Server wird in
DCODCF-Rack14 bereitgestellt; der Hardware-\ac{RAID}-Controller
wird auf \ac{RAID}\,1 für das System und \ac{RAID}\,5 über
6$\times$4\,TB HDDs für den Datastore konfiguriert. Zusätzlich
wird eine \ac{SFP}-Netzwerkkarte mit 2$\times$10\,GbE eingebaut
und per \ac{LACP} zu einem logischen Interface gebündelt
(Aufwand ca.\ 10\,min); die Verkabelung läuft über die Patchpanels
in Rack14/16 zum Cisco-Switch
\texttt{aci-dc03-og2-r16-aly-2153}. Parallel wird das aktuelle
\ac{PBS}-4-ISO als bootfähiges Installationsmedium vorbereitet und
mit dem \ac{DKFZ}-Netzwerkteam der Anschluss an das
\emph{server}-\ac{VLAN} sowie die Firewall-Freischaltungen zum
\ac{PVE}-Cluster abgestimmt.

Die endgültige LACP-Konfiguration auf dem \ac{PBS} zeigt das
folgende Listing:
\begin{lstlisting}[caption={\texttt{/etc/network/interfaces} auf odcf-fs04}]
auto lo
iface lo inet loopback

iface nic0 inet manual
iface nic1 inet manual
iface nic2 inet manual
iface nic3 inet manual

auto nic4
iface nic4 inet manual

auto nic5
iface nic5 inet manual

auto bond0
iface bond0 inet static
        address 10.131.196.78/22
        gateway 10.131.196.97
        bond-slaves nic4 nic5
        bond-miimon 100
        bond-mode 802.3ad
        bond-xmit-hash-policy layer3+4
        bond-lacp-rate fast

source /etc/network/interfaces.d/*
\end{lstlisting}


\subsection{Installation des Proxmox Backup Servers}
\label{sec:InstallationPBS}
Die Installation erfolgt direkt vom \ac{PBS}-4-ISO. Nach Abschluss des
geführten Installers wird das System wie folgt grundkonfiguriert:
\begin{itemize}
    \item Hostname \texttt{odcf-fs04} mit der IP \texttt{10.131.196.78}
          im \emph{server}-\ac{VLAN}; die Web-Oberfläche ist unter
          \url{https://10.131.196.78:8007} erreichbar.
    \item Eintrag im internen \ac{DKFZ}-DNS unter dem o.\,g.\
          Hostnamen.
    \item Konfiguration des \texttt{pbs-no-subscription}-
          Repositories, da keine Enterprise-Subscription beschafft
          wird; anschließend vollständiges \texttt{apt}-Upgrade.
    \item NTP-Synchronisation gegen die internen Zeitserver des
          \ac{DKFZ}.
\end{itemize}
Tatsächlicher Zeitaufwand für Installation und Grundkonfiguration:
ca.~5\,h.


\subsection{Konfiguration des Datastores}
\label{sec:Datastore}
Auf dem \ac{RAID}\,5-Volume wird ein \texttt{ext4}-Dateisystem
angelegt und unter \texttt{/mnt/datastore/backup} eingehängt.
Anschließend wird im \ac{PBS} ein Datastore mit dem Namen
\texttt{backup} angelegt, der auf dieses Verzeichnis verweist.
Die Konfiguration findet sich in \texttt{datastore.cfg}:
\begin{lstlisting}[caption={\texttt{/etc/proxmox-backup/datastore.cfg}}]
datastore: backup
        path /mnt/datastore/backup
\end{lstlisting}

Auf dem Datastore wird zusätzlich ein wöchentlicher Verify-Job
angelegt, der die Prüfsummen aller gespeicherten Chunks gegen die
Metadaten verifiziert.


\subsection{Anbindung an das DKFZ-LDAP}
\label{sec:LDAPAnbindung}
Für die Anmeldung regulärer Administratorinnen und Administratoren
wird der \ac{PBS} an das \ac{DKFZ}-\ac{LDAP} angebunden.
Dazu wird im \ac{PBS} ein \ac{LDAP}-Realm angelegt; berechtigt sind
Benutzer der \ac{AD}-Gruppe \texttt{ODCF-SysAdmins}.
Bestehende Benutzer können sich anschließend mit ihren persönlichen
Konten an der Web-Oberfläche anmelden.
Der lokale \texttt{root}-Account bleibt für die Initialkonfiguration
und Notfallzugriffe erhalten.

Die resultierende Realm-Konfiguration zeigt das folgende Listing:
\begin{lstlisting}[caption={\texttt{/etc/proxmox-backup/domains.cfg}}]
pam: pam
        comment Linux PAM standard authentication
        type pam

pbs: pbs
        comment Proxmox Backup authentication server
        type pbs

ad: dkfz-heidelberg.de
        base-dn DC=ad,DC=dkfz-heidelberg,DC=de
        bind-dn ldap@ad.dkfz-heidelberg.de
        default true
        filter (&(objectClass=user)(memberOf=CN=ODCF-SysAdmins,OU=Kst-W610,OU=Fsp-W,OU=DKFZ,DC=ad,DC=dkfz-heidelberg,DC=de))
        mode ldaps
        port 636
        server1 ldap1.dkfz-heidelberg.de
        sync-attributes email=mail,firstname=givenName,lastname=sn
        user-classes user
\end{lstlisting}

Tatsächlicher Zeitaufwand für die LDAP-Einrichtung: ca.~0{,}5\,h.


\subsection{Integration in den PVE-Cluster}
\label{sec:PVEIntegration}
Für die Anbindung des \ac{PBS} an den \ac{PVE}-Cluster wird ein
dedizierter \ac{API}-Token im \ac{PBS} erstellt. Dieser Token erhält
ausschließlich die für Backups und Verifizierung notwendigen
Berechtigungen auf dem entsprechenden Datastore.

Im \ac{PVE} wird der \ac{PBS} anschließend als Storage vom Typ
\emph{Proxmox Backup Server} hinzugefügt; dabei werden Server-Adresse
(\texttt{odcf-fs04} / \texttt{10.131.196.78}), Datastore-Name,
\ac{API}-Token sowie der TLS-Fingerprint des \ac{PBS}-Zertifikats
hinterlegt.
Ein Test-Backup einer unkritischen \ac{VM} verifiziert die korrekte
Anbindung, bevor produktive Jobs aufgesetzt werden.

Tatsächlicher Zeitaufwand für die PVE-Anbindung: ca.~0{,}5\,h.


\subsection{Einrichtung der Backup-Jobs}
\label{sec:BackupJobs}
Die produktiven Backup-Jobs werden im \ac{PVE} pro \ac{VM}-Klasse
angelegt; sowohl für kritische als auch für Test-\acp{VM} gilt:
zwei Sicherungen pro Tag, Prune-Regel \texttt{keep-last=3}.
Die Jobs werden zunächst manuell ausgelöst, um Laufzeiten und
Bandbreitenbedarf zu beobachten; anschließend werden sie auf den
geplanten Zeitplan umgestellt.


\subsection{Monitoring und Benachrichtigungen}
\label{sec:Monitoring}
Für Fehlerbenachrichtigungen wird der eingebaute E-Mail-Mechanismus
des \ac{PBS} konfiguriert (SMTP-Relay des \ac{DKFZ}).
Zusätzlich wird der Server in das bestehende \ac{Prometheus}-
Monitoring der \ac{ODCF} aufgenommen; der \ac{PBS} stellt dafür
einen nativen Metrik-Endpunkt bereit, der per \texttt{node\_exporter}
bzw.\ \texttt{proxmox-backup-exporter} angebunden wird.
Überwacht werden insbesondere:
\begin{itemize}
    \item Erreichbarkeit und Auslastung des \ac{PBS}.
    \item Belegung des Datastores (Vermeidung eines vollen Pools).
    \item Status der Backup- und Verify-Jobs.
    \item \ac{RAID}-Status des Hardware-Controllers.
\end{itemize}
